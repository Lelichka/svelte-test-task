Реализовано:

- Загрузка клиентов с обработкой ошибок и состояния loading 
- Нормализация данных 
  - Преобразование дублирующихся id
  - Преобразование баланса к числу
  - Преобразование даты к числу timestamp (некорректные даты -> null)
  - Определение VIP-статуса
- Таблица клиентов
  - Подсветка VIP-клиентов
  - Подсветка отрицательного баланса
  - Отображение некорректных дат в виде "-"
- Фильтрация по статусу
- Поиск по имени
- Сортировка по balance и createdAt

Не реализовано:

- Пагинация
- Действия с клиентами (редактирование/удаление)
- Адаптивный дизайн (изменение вида таблицы для разных размеров устройств)

Assumptions:

- В API могут приходить клиенты с одинаковыми id.
Запись с дублирующимся id не удаляется, ее id преобразуется в уникальный с помощью счетчика (чтобы видеть исходный id)
- Баланс может быть отрицательным или null, строковым числом или "Infinity".
Отрицательный баланс возможен.
Преобразуем баланс к числу, null или некорректные значения к 0.
- Дата создания может быть некорректной или null.
Преобразуем корректные даты в timestamp, некорректные к null. 

Что можно было бы улучшить:

- Добавление пагинации
- Возможность действий с клиентами
- Реализация адаптивного дизайна
